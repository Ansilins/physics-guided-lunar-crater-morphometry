About Lunar Crater Detection

This project focuses on detecting and analyzing lunar craters using advanced computer vision techniques, with a special emphasis on using the shadow method to calculate the depth of craters in 2D images. By leveraging shadow-based and DEM (Digital Elevation Model) methods, the tool provides detailed measurements of crater diameter, depth, and volume, enabling a comprehensive comparison of detection techniques.

How to Use This Site

Follow these steps to use the Lunar Crater Detection tool:

  • Step 1: Navigate to the Extract Page - Upload a GeoTIFF file containing lunar surface data. The tool will extract the necessary bands (PNG for visualization and TIFF for elevation data).
  • Step 2: Calculate Solar Geometry - Input the observer and sub-solar coordinates (latitude and longitude) to calculate the solar elevation and azimuth. These values are crucial for shadow method depth calculations.
  • Step 3: Proceed to the Input Page - Upload the extracted PNG and TIFF files (or use new ones). Adjust the sun elevation and azimuth if needed, then run the detection process.
  • Step 4: View Detection Results - On the Input page, view the detected craters with their measurements (center coordinates, diameter, depth, and volume) for both Shadow and DEM methods.
  • Step 5: Analyze Results - Go to the Results page to see a comparison of the Shadow and DEM methods through charts (diameter, depth, and volume) and download the detailed CSV file.

Note: There are multiple types of crater shapes, and this tool cannot accurately predict the depth of all crater types using the shadow method. We plan to address this in further research.

Limitations

While this tool is powerful, it has certain limitations:

  • Input File Requirements - The tool requires specific GeoTIFF files with appropriate bands (band 1 for PNG visualization, band 6 for elevation data). Files that do not meet these requirements may cause errors.
  • Image Dimensions - The input images must be 500x500 pixels. Images of other sizes will result in an error during processing.
  • Solar Geometry Accuracy - The accuracy of shadow method depth calculations depends on correct solar geometry inputs. Incorrect coordinates may lead to inaccurate results.
  • Crater Detection - The tool may miss small craters or fail to pair craters between Shadow and DEM methods if the confidence thresholds are not met or if shadows are not well-defined.

Dataset Sources

The datasets used for lunar crater detection can be obtained from the following sources:

  • Primary Dataset - The primary dataset is sourced from QuickMap. This dataset uses the Equidistant Cylindrical (< 75 deg.) projection scale to calculate azimuth and elevation and to select the region for analysis. We will work more on this to add support for all projections in future updates.
  • Exploration and Analysis - To select a location or region for analysis, use this link: QuickMap Projections.
  • Kaggle Dataset - The refined dataset, including calculated azimuth and elevation data in the CSV and GeoTIFF images, is available on Kaggle: MoonFinal991. Users must note the observer latitude and longitude from this dataset for accurate analysis.
